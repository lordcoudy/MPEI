2.3.2.	Генерация Главного Ключа
Для генерации Главного Ключа необходимо выполнить следующие действия:
1.	Выбрать команду "Создание Главного Ключа". 
2.	В поле "Имя Главного Ключа" ввести имя файла. Стандартное имя для Главного Ключа - gk.db3, именно такой файл загружает устройство шифрования при инициализации. Имя, предлагаемое по умолчанию, можно изменить, но перед использованием этот ГК придется все равно переименовать к стандартному виду.
3.	В поле "Каталог хранения Главного Ключа" указать существующий каталог, в котором будет расположен созданный ГК. В данном поле можно использовать каталоги на дисках и на смарт-картах.
4.	Флажок "Шифровать на пароле" определяет, будет ли Главный Ключ зашифровываться на пароле. Этот пароль в последствии будет запрашиваться при инициализации устройства шифрования созданным Главным Ключом.
5.	Переключателем "Получение Узла Замены" выбрать способ получения Узла Замены для генерации ГК. Если выбран вариант "Использовать текущий, загруженный в Криптон, УЗ", то для генерации используется загруженный в Криптон УЗ. Если выбран вариант "Использовать УЗ из файла", происходит поиск указанного файла и использование этого УЗ.
6.	Нажать кнопку "Создать".
Если в указанном каталоге уже существует одноименный файл ГК, то он будет переименован (расширение изменено на .old).
